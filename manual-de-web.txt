1. VISIÓN GENERAL DEL PROYECTO

Arquitectura y objetivos del sitio

1.1 Descripción del sitio

El sitio es la presencia digital oficial del dojo de artes marciales.
Está compuesto por dos partes principales:

- Sitio público (frontend): visible para todos los visitantes, con
  información del dojo, clases, shop y formularios.

- Panel Administrativo (admin): área privada donde el dueño puede
  gestionar todo el contenido sin tocar código.

> *💡 El sitio fue diseñado para que el administrador tenga control
> total del contenido. La idea es que cualquier texto, imagen o producto
> pueda actualizarse desde el panel sin necesidad de un desarrollador.*

1.2 Inspiración visual

El diseño sigue la estética del sitio Gymbox.com: oscuro, impactante,
con grandes imágenes hero, carruseles de contenido y tipografía en
mayúsculas. Adaptado para un dojo de artes marciales.

- Fondo base: blanco (#FFFFFF) y gris claro (#F2F2F2)

- Color primario: Rojo #D10A0A (acción, botones, títulos destacados)

- Color secundario: Dorado #C9A84C (jerarquía, rangos, detalles premium)

- Color oscuro: #262626 (header, footer, textos principales)

- Tipografía: Sans-serif bold en mayúsculas para headings (ej: Inter,
  Bebas Neue u Oswald)

1.3 Principios de diseño

- Consistencia visual: todas las páginas comparten el mismo header,
  footer y paleta.

- Mobile-first: diseño pensado desde el móvil hacia arriba.

- Simplicidad del admin: el panel no es Elementor. Tiene bloques
  predefinidos simples.

- WhatsApp como canal principal: inscripciones, compras y consultas van
  a WhatsApp.

5. PANEL ADMINISTRATIVO

Diseño y funcionalidades del admin

5.1 Acceso y autenticación

El admin utiliza Supabase Auth. Flujo:

9.  Admin va a /admin/login

10. Ingresa email + password

11. Supabase Auth verifica credenciales

12. Si es válido, redirige a /admin (dashboard)

El middleware.ts de Next.js protege todas las rutas /admin/*. Si no hay
sesión activa, redirige a /admin/login.

> // middleware.ts
>
> import { createMiddlewareClient } from
> '@supabase/auth-helpers-nextjs'
>
> export async function middleware(req) {
>
> const res = NextResponse.next()
>
> const supabase = createMiddlewareClient({ req, res })
>
> const { data: { session } } = await supabase.auth.getSession()
>
> if (!session && req.nextUrl.pathname.startsWith('/admin')) {
>
> return NextResponse.redirect(new URL('/admin/login', req.url))
>
> }
>
> return res
>
> }

6. PAGE BUILDER

Sistema de bloques de contenido

6.1 Filosofía del page builder

El page builder NO es Elementor ni un editor libre. Es un sistema de
bloques predefinidos con configuración simple. Cada página tiene una
lista de bloques ordenados que se pueden:

- Agregar (elegir de una lista de tipos)

- Reordenar (drag & drop)

- Editar (formulario específico de ese bloque)

- Ocultar / mostrar (toggle sin borrar)

- Eliminar

> **⚠️ No es un editor de píxeles. El admin configura el contenido
> (texto, imagen, color de fondo, etc.), pero el diseño visual lo define
> el desarrollador en el código del componente.**

6.2 Tipos de bloques disponibles

**Bloque: BANNER**

Ocupa el 100% del ancho de la pantalla. Configuración:

- Imagen de fondo (subida desde Storage)

- Texto principal (título grande)

- Texto secundario (subtítulo)

- Color de overlay (por defecto semi-transparente negro)

- Botón 1: texto + URL (opcional)

- Botón 2: texto + URL (opcional)

- Altura: 'full' (100vh) o 'medium' (60vh) o 'small' (40vh)

Estructura JSON del bloque:

> { "type": "banner",
>
> "content": {
>
> "image_url": "https://...",
>
> "title": "KARATE-DO",
>
> "subtitle": "El camino del puño vacío",
>
> "overlay_opacity": 0.5,
>
> "height": "full",
>
> "btn1_text": "Inscribirse",
>
> "btn1_url": "/inscripciones",
>
> "btn2_text": "Ver horarios",
>
> "btn2_url": "#horarios"
>
> }
>
> }

**Bloque: TEXTO**

Sección de texto enriquecido. Configuración:

- Título de sección (opcional)

- Contenido HTML (editor rich text tipo TipTap o ReactQuill)

- Alineación: izquierda / centro / derecha

- Fondo: blanco / gris claro / oscuro

> { "type": "text",
>
> "content": {
>
> "title": "Sobre el dojo",
>
> "body": "<p>El dojo fue fundado...</p>",

>

**Bloque: IMAGEN**

Imagen standalone con caption. Configuración:

- Imagen (subida desde Storage)

- Caption (texto debajo, opcional)

- Ancho: 'full' / 'contained' (máx 800px)

**Bloque: IMAGEN + TEXTO (lado a lado)**

Grid de dos columnas: imagen a la izquierda, texto a la derecha (o
viceversa). Configuración:

- Imagen

- Lado de la imagen: izquierda / derecha

- Título

- Párrafo de texto

- Botón (opcional): texto + URL

**Bloque: GALERÍA**

Grid de imágenes con lightbox. Configuración:

- Categoría de galería a mostrar (o 'todas')

- Cantidad máxima de imágenes a mostrar (6, 9, 12)

- Mostrar botón 'Ver más' (sí/no) → link a /galeria o a galería
  filtrada

**Bloque: CARRUSEL**

Slider horizontal de tarjetas con imagen. Configuración:

- Items del carrusel: cada item tiene imagen, título, subtítulo, link

- Autoplay: sí/no + velocidad

- Flechas de navegación: sí/no

**Bloque: STATS / DESTACADOS**

Fila de métricas visuales. Configuración:

- Lista de stats: cada stat tiene número/valor y etiqueta (ej: '200+
  Alumnos', '15 Años')

- Fondo: blanco / oscuro / rojo

**Bloque: PRODUCTOS DESTACADOS**

Grid con los productos marcados como destacados del shop. Sin
configuración especial --- toma automáticamente los productos activos.

**Bloque: FORMULARIO DE INSCRIPCIÓN**

Incrusta el formulario de inscripción en cualquier página. Sin
configuración (usa los datos globales de WhatsApp).

**Bloque: BOTÓN WHATSAPP**

CTA grande con botón de WhatsApp. Configuración:

- Texto principal (ej: '¿Listo para empezar?')

- Subtexto (opcional)

- Mensaje de WhatsApp a enviar

- Fondo: blanco / rojo / oscuro

**Bloque: EMBED VIDEO**

Embed de YouTube. Configuración:

- URL del video de YouTube

- Título del bloque (opcional)

**Bloque: MAPA**

Incrusta Google Maps. Configuración:

- URL de Google Maps Embed (se obtiene desde Google Maps > Compartir >
  Insertar mapa)

- Altura del mapa en px

6.3 Implementación del Page Builder

El page builder en la ruta /admin/pages/[id] tiene:

13. Panel izquierdo: lista de bloques actuales de esa página (ordenables

El drag & drop actualiza el campo order_index de cada bloque en
Supabase.

> // Ejemplo de guardar orden después de drag & drop
>
> const reorderedBlocks = blocks.map((block, index) => ({
>
> ...block,
>
> order_index: index
>
> }))
>
> await supabase.from('page_blocks').upsert(reorderedBlocks)

7. GESTIÓN DEL SHOP

Productos con integración WhatsApp

7.1 ABM de productos

Desde /admin/shop el admin puede crear, editar y eliminar productos. El
formulario de producto tiene:

- Nombre del producto

- Precio (número con 2 decimales)

- Descripción (rich text básico)

- Imagen (subida a Supabase Storage, bucket 'product-images')

- Talles disponibles (checkbox: XS, S, M, L, XL, XXL, Única talla)

- Activo / Inactivo (toggle)

- Destacado (toggle --- aparece en el bloque 'Productos Destacados'
  del page builder)

7.2 Generación del link de WhatsApp

La función whatsapp.ts genera el link dinámicamente:

> // lib/whatsapp.ts
>
> export function buildWhatsAppLink({
>
> phone, // ej: '5493514XXXXXXX'
>
> productName, // ej: 'Remera Karate blanca'
>
> size, // ej: 'M'
>
> }: WhatsAppProductParams): string {
>
> const msg = size
>

11.2 Renderizador de bloques

> // components/site/BlockRenderer.tsx
>
> import { BannerBlock } from './blocks/BannerBlock'
>
> import { TextBlock } from './blocks/TextBlock'
>
> import { GalleryBlock } from './blocks/GalleryBlock'
>
> // ... otros bloques
>
> export function BlockRenderer({ block }: { block: PageBlock }) {
>
> if (!block.is_visible) return null
>
> switch (block.block_type) {
>
> case 'banner': return <BannerBlock content={block.content} />
>
> case 'text': return <TextBlock content={block.content} />
>
> case 'gallery': return <GalleryBlock content={block.content} />
>
> case 'products': return <ProductsBlock content={block.content} />
>
> case 'form': return <RegistrationForm />
>
> case 'whatsapp_cta': return <WhatsAppCTA content={block.content}
> />
>
> default: return null
>
> }
>
> }

11.3 Página pública que carga bloques

> // app/(site)/home/page.tsx
>
> import { createServerClient } from '@/lib/supabase/server'
>
> import { BlockRenderer } from '@/components/site/BlockRenderer'
>
> export default async function HomePage() {
>
> const supabase = createServerClient()
>
> const { data: page } = await supabase
>
> .from('pages')
>
> .select('id')
>
> .eq('slug', 'home')
>
> .single()
>
> const { data: blocks } = await supabase
>
> .from('page_blocks')
>
> .select('*')
>
> .eq('page_id', page?.id)
>
> .order('order_index')
>
> return (
>
> <main>
>
> {blocks?.map(block => (
>
> <BlockRenderer key={block.id} block={block} />
>
> ))}
>
> </main>
>
> )
>
> }
